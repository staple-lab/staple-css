BEST PRACTICES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DO's:
✓ Use token scales (space, color, typography)
✓ Compose with Box, Flex, Column, Row, Grid
✓ Use semantic tokens (primary, danger, success, warning, neutral)
✓ Full TypeScript type safety
✓ Keyboard navigation support
✓ WCAG AA color contrast
✓ Test with screen reader
✓ Respect prefers-reduced-motion

DON'Ts:
✗ Raw pixel values (avoid hardcoding 16px, 32px, etc.)
✗ Custom padding/margin (use space tokens)
✗ Hardcoded colors (use semantic colors)
✗ Ignore TypeScript warnings
✗ Skip focus states
✗ Use color alone for meaning
✗ Assume desktop-only
✗ Ignore motion preferences

IMPLEMENTATION PATTERNS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Pattern 1: Email Card
- Use Card primitive with shadow-1
- Column layout for vertical stacking
- Row layout for footer buttons
- Apply space tokens for gap spacing
- Use tone="muted" for secondary text

Pattern 2: Login Form
- Semantic form element
- Label elements with htmlFor association
- Input elements with proper type attributes
- Fieldset for grouped inputs (optional)
- Button with type="submit"

Pattern 3: Dashboard Metrics
- Grid layout with auto-fill columns
- Card primitive for each metric
- Flex for label + icon alignment
- Conditional coloring based on trend
- Responsive: 1 column on mobile, 2-4 on desktop
