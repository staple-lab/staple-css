   Key: Respect user motion preferences. Use motion tokens for consistency.

5. Responsive Design
   • Mobile-First Approach
   • Breakpoint System
   • Touch Target Size (44px minimum)
   Key: Design for small screens first. Use intrinsic responsive patterns.

6. Accessibility (a11y)
   • Semantic HTML
   • Color Contrast
   • Focus Management
   • Keyboard Navigation
   Key: Build accessibility in from the start. Test with keyboard and screen reader.

7. Component Patterns
   • Composition Over Inheritance
   • Props API Design
   • Escape Hatches (className)
   Key: Compose primitives together. Use className for exceptions, not rules.

8. Design Tokens Strategy
   • Token Hierarchy
   • Naming Conventions
   • Versioning & Updates
   Key: Maintain token consistency. Document changes. Communicate updates to team.


GUIDE DETAILS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

**Guide 1: Spacing & Layout**
- Use space-0 through space-8 scale
- Avoid custom pixel values like 13px or 25px
- Always use padding from tokens, not margin for initial spacing
- Grid gap should use space tokens
- Max-width for readability: 65ch (~900px)

**Guide 2: Color & Contrast**
- Semantic colors: primary, danger, success, warning, neutral
- Test with WebAIM contrast checker
- Support light and dark themes
- Use color + icon/text for meaning, not color alone

**Guide 3: Typography & Text**
- Font sizes 0-6 scale (12px to 32px)
- Line height 1.25, 1.5, or 1.75
- Body text max width 65 characters
- Use weight 400 (regular) for body, 600+ for headings

**Guide 4: Motion & Animation**
- Duration scale: 75ms (instant) to 1000ms (slowest)
- Easing: ease-out for interactions
- Default motion support only. Disabled when prefers-reduced-motion

**Guide 5: Responsive Design**
- Breakpoints: base (0px), sm (640px), md (768px), lg (1024px), xl (1280px), 2xl (1536px)
- Mobile-first CSS: base styles first, then media queries up
- Touch targets minimum 44x44px

**Guide 6: Accessibility**
- Use semantic HTML: main, section, nav, article, aside
- Visible focus states (2px outline)
- Icon buttons need aria-label
- Decorative elements need aria-hidden

**Guide 7: Component Patterns**
- Always compose from Box, Flex, Grid, Text, Card primitives
- Props are constrained to token keys
- Use className for exceptions
- Test keyboard navigation (Tab, Enter, Escape)

**Guide 8: Design Tokens Strategy**
- Tokens are the source of truth
- Update token, all components reflect change
- Use version control for token changes
- Document breaking changes in CHANGELOG
